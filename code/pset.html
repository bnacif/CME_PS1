function clean_currency(val)
    if ismissing(val) || val == ""
        return 0.0
    elseif val isa Number
        return Float64(val)
    else
        # remove R$, thousands separator and substitute ',' for '.'
        s = replace(string(val), r"R\$|\." => "", "," => ".")
        return parse(Float64, s)
    end
end

(a)

What is the relationship between the prize value announced and the revenue collected? What is the elasticity?

Now we start the analysis. To estimate the relationship between the announced prize and the total revenue collected, we regress the log of total revenue on the log of the announced prize. In order to do that, we consider the prize variable lagged in one period, since the prize value announced is located in the previous row relative to its draw. We also restrict the sample to draws from June 2009 onwards, since there is no positive revenue information before that.

Code 
# lag prize
df[!, :prize_ann] = lag(df.prize);

# filter for positive values of prize
df = filter(row -> 
    row.date >= Date(2009, 6, 1) && # missing data before
    row.revenue > 0 && 
    !ismissing(row.prize_ann) && 
    row.prize_ann > 0, 
    df
);

# regression
df[!, :log_revenue] = log.(df.revenue);
df[!, :log_prize] = log.(df.prize_ann);

reg = lm(@formula(log_revenue ~ log_prize), df);  # intercept added by default

println("The elasticity between the prize announced and the revenue collected is: ", round(coef(reg)[2], digits=3))
The elasticity between the prize announced and the revenue collected is: 0.401

The prize elasticity of the revenue, given by the regression coefficient, is 0.4. This means that a 1% increase in the announced prize implies an increase of 0.4% in the total revenue collected.

(b)

Estimating the number of players.

We estimate the number of tickets sold in each draw by dividing total revenue by the ticket price. For each game \(j\), the number of participants (\(N_j\)) is: \[ N_j = \frac{rev_j}{p_j} \]

where \(rev_j\) is the revenue and \(p_j\) is the ticket price.

Note that we assume all individuals choose the cheapest option and buy only one ticket. This is a fairly strong assumption given that players can choose among several options to participate on the lottery. Still, this gives us a rather good approximation to start with.

Besides, since the ticket price has changed several times over the years, we define a function that returns the correct historical price for each draw date and apply it to the dataset. The source of the data is detailed in others/price_changes.

Code 
function get_price(data::Date)
    if data < Date(2014, 5, 11)
        return 2.00
    elseif data < Date(2015, 5, 24)
        return 2.50
    elseif data < Date(2019, 11, 10)
        return 3.50
    elseif data < Date(2023, 4, 30)
        return 4.50
    elseif data < Date(2025, 7, 10)
        return 5.00
    else
        return 6.00
    end
end

# adding column of prices
df[!, :preco] = get_price.(df.date);

# N_j = revenue / price
df[!, :n_tickets] = df.revenue ./ df.preco;

(c)

Expected prize for each draw.

The expected prize per ticket is computed as the total amount paid out across all prize tiers - 4, 5, 6 correct numbers - divided by the total number of tickets sold in that draw. In math terms,

\[ \mathbb{E}[Y_j] = \frac{G6_j}{N_j}Y^{6}_{j} + \frac{G5_j}{N_j}Y^{5}_{j} + \frac{G4_j}{N_j}Y^{4}_{j} \]

where \(G\theta_j\) is the number of individuals who won one of the prizes, depending on the number \(\theta \in \{4,5,6\}\) of numbers they got right, and \(Y_j^{\theta}\) is the corresponding prize paid to each winner.

Code 
# G6/Nj * Y6 + G5/Nj * Y5 + G4/Nj * Y4
df[!, :total_prize] = (
    (df.ganhadores_6_acertos .* df.rateio_6_acertos) .+ 
    (df.ganhadores_5_acertos .* df.rateio_5_acertos) .+ 
    (df.ganhadores_4_acertos .* df.rateio_4_acertos)
)

# dividing by N_j
df[!, :exp_prize] = df.total_prize ./ df.n_tickets;

println("The expected prize across draws is: ", mean(df.exp_prize))
The expected prize across draws is: 0.9129613481870189

(d)

Expected return of buying a ticket.

Finally, we compute the expected return. To do so, we divide the expected prize by the ticket price: \[ R_j = \frac{\mathbb{E}[Y_j]}{p_j} \]

Code 
# R = E[Y_j]/p_j
df[!, :exp_return] = df.exp_prize ./ df.preco;

println("The expected return across draws is: ", mean(df.exp_return))
The expected return across draws is: 0.2571308530606728

(e)

Analyzing the average return for a given prize announcement.

The final step is to evaluate the average return for a given prize announcement: \[ \mathbb{E}[R_j | P_j] \]

The unconditional expected return considering all games after 2009 is 0.26. This means that, on average, each R$1 invested in a ticket returns only 26 cents.

Since this is only an average result, let us now evaluate the dispersion of expected returns given the announced prize. In the graph below, each dot is a draw. We restrict the range of prize values to highlight the underlying relationship.

Therefore, we adopt a simpler strategy. Below, we group the draws according to prize (ranges of R$25 million) and calculate the average return for each bin. The resulting graph gives an idea of the relationship between the variables.

Code 
# bin strategy

Overall, it seems that the function of expected return given the announced prize is somewhat concave. At first, as the prize increases, return increases, since total revenue expands faster than the number of players. However, once the number of participants is “high enough”, individual expected gains fall considerably.

The histogram shows how right skewed the distribution of prizes is, which is explained mostly by the highest prizes in Mega da Virada. This helps account for the dispersion of draws we found before - there are very few observations with the highest prizes.

What is clear is that there is no range of announced prize presenting an average return above one.
